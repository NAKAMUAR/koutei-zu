## 1. 画面・モーダル一覧

### 1.1 画面（ビュー）

| 画面ID | 画面名 | コンポーネント | 遷移トリガ |
|---|---|---|---|
| S-LOGIN | サインイン | App（認証前） | 初回アクセス／サインアウト後 |
| S-INPUT | 入力 | InputView | ナビ「入力」 |
| S-CAL | カレンダー | CalendarView | ナビ「カレンダー」 |
| S-ASG | 担当者別 | AssigneeView | ナビ「担当者別」 |
| S-MSG | サマリー | MessageView | ナビ「サマリー」 |
| S-DONE | 完了 | DoneView | ナビ「完了」 |
| S-MST | マスタ | MasterView | ナビ「マスタ」 |

### 1.2 モーダル・パネル（画面上にオーバーレイ表示）

| ID | 名称 | コンポーネント | 起動元 |
|---|---|---|---|
| M-SET | 設定パネル | Header内 | ⚙ボタン |
| M-QUOTE | 過去案件引用 | QuoteModal | S-INPUT「過去案件から引用」 |
| M-CONFIRM | 確認（開始移動／納期超過） | ConfirmModal | 登録/更新時の試算結果 |
| M-COMPLETE | 完了ダイアログ | CompleteDialog | 視点完了／案件完了 |
| M-ENDPROMPT | 終了超過ポップアップ | EndPromptModal | 視点の終了予定超過を検知（自動） |

## 2. 全体画面遷移図

```mermaid
stateDiagram-v2

    S_LOGIN --> S_INPUT : サインイン成功（許可メール）
    S_LOGIN --> S_LOGIN : 非許可メール（自動サインアウト）

    state "アプリ本体（認証後）" as APP {
        S_INPUT --> S_CAL : ナビ「カレンダー」
        S_INPUT --> S_ASG : ナビ「担当者別」
        S_INPUT --> S_MSG : ナビ「サマリー」
        S_INPUT --> S_DONE : ナビ「完了」
        S_INPUT --> S_MST : ナビ「マスタ」

        S_CAL --> S_INPUT : ナビ「入力」
        S_ASG --> S_INPUT
        S_MSG --> S_INPUT
        S_DONE --> S_INPUT
        S_MST --> S_INPUT

        S_CAL --> S_ASG
        S_ASG --> S_CAL
        S_DONE --> S_MST
        note right of S_INPUT
            どの画面からも
            ナビで相互遷移可能
        end note
    }

    APP --> S_LOGIN : サインアウト

    S_INPUT : 入力（登録フォーム＋進行中一覧）
```

> ナビゲーションバーは全画面共通でヘッダーに常駐し、6画面は相互に直接遷移できる。上図は代表的な経路のみを示す。

## 3. 入力画面（S-INPUT）の詳細遷移

入力画面は「登録フォーム」と「進行中タスク一覧」を持ち、編集・引用・各種操作の起点となる。

```mermaid
flowchart TD
    INPUT["S-INPUT 入力画面"]

    INPUT -->|"過去案件から引用"| QUOTE["M-QUOTE 引用モーダル"]
    QUOTE -->|"案件を選択"| INPUT
    QUOTE -->|"閉じる"| INPUT

    INPUT -->|"登録/更新ボタン"| CHECK{"試算<br/>simulateFormSchedule"}
    CHECK -->|"開始移動 or 納期超過あり"| CONFIRM["M-CONFIRM 確認モーダル"]
    CHECK -->|"問題なし"| SAVE[("保存 → Firestore")]
    CONFIRM -->|"このまま登録"| SAVE
    CONFIRM -->|"戻る"| INPUT
    SAVE --> INPUT

    INPUT -->|"ステップ編集"| EDIT_STEP["編集モード: step"]
    INPUT -->|"視点編集"| EDIT_VP["編集モード: viewpoint"]
    INPUT -->|"案件を編集"| EDIT_PRJ["編集モード: project"]
    INPUT -->|"視点を追加"| ADD_VP["視点追加フォーム"]
    INPUT -->|"ステップ追加"| ADD_STEP["ステップ追加フォーム"]
    EDIT_STEP --> INPUT
    EDIT_VP --> INPUT
    EDIT_PRJ --> INPUT
    ADD_VP --> INPUT
    ADD_STEP --> INPUT

    INPUT -->|"視点完了 / 案件完了"| COMPLETE["M-COMPLETE 完了ダイアログ"]
    COMPLETE -->|"完了する（実終了時刻入力）"| DONE_VIEW["S-DONE 完了へ遷移"]
    COMPLETE -->|"キャンセル"| INPUT

    INPUT -->|"案件中止"| CANCEL{"確認"}
    CANCEL -->|"OK"| DONE_VIEW
    CANCEL -->|"キャンセル"| INPUT
```

> 編集の3スコープ（step / viewpoint / project）はいずれも同一フォームに展開され、保存スコープが切り替わる。フォーム上部の見出しで現在の編集対象を表示する。

## 4. 終了超過ポップアップ（M-ENDPROMPT）の遷移

このポップアップは利用者操作ではなく、**1分ごとの再評価で視点の終了予定超過を検知すると自動表示**される。

```mermaid
flowchart TD
    TICK["1分ごとの now 更新"] --> EVAL{"終了予定を過ぎた<br/>視点があるか<br/>（スヌーズ除く）"}
    EVAL -->|"なし"| HIDE["非表示"]
    EVAL -->|"あり"| PROMPT["M-ENDPROMPT 表示<br/>（視点単位）"]

    PROMPT -->|"① 視点完了"| C1["実終了時刻入力 → done化"]
    PROMPT -->|"② 追加修正"| C2["追加ステップを付与"]
    PROMPT -->|"③ 遅延"| C3["新終了予定へ更新<br/>差分稼働を加算"]
    PROMPT -->|"④ 後で"| C4["30分スヌーズ"]

    C1 --> DONEVIEW["S-DONE へ遷移"]
    C2 --> RECALC["再スケジュール"]
    C3 --> RECALC
    C4 --> HIDE
    RECALC --> EVAL
```

## 5. カレンダー画面（S-CAL）の内部遷移

カレンダーは表示モードの切替を持ち、ブロッククリックで入力画面（案件編集）へ遷移する。

```mermaid
flowchart LR
    CAL["S-CAL カレンダー"]
    CAL -->|"表示モード切替"| MODE["1日 / 全期間"]
    MODE --> CAL
    CAL -->|"前/翌/今日"| NAV["期間ナビゲーション"]
    NAV --> CAL
    CAL -->|"タスクブロックをクリック"| EDIT["S-INPUT（案件編集フォーム）"]
    CAL -->|"ブロックをドラッグ&ドロップ"| REORDER["案件順を変更（自画面更新）"]
    CAL -->|"担当者名をドラッグ"| ROWREORDER["担当者表示順を変更"]
```

## 6. マスタ画面（S-MST）の構成

マスタ画面は単一スクロール内に5セクションを縦配置する（画面遷移ではなくセクション内編集）。

```mermaid
flowchart TD
    MST["S-MST マスタ画面"]
    MST --> SEC1["お客様マスタ（会社＋担当者）"]
    MST --> SEC2["従業員マスタ（表示順管理）"]
    MST --> SEC3["残業の登録（OvertimeManager）"]
    MST --> SEC4["欠勤・休日・不在の登録（AbsenceManager）"]
    MST --> SEC5["会社の表示順（CompanyOrderView）"]
```

> 旧「表示順設定」タブと、設定パネル内にあった残業・欠勤登録は、本画面に集約済み。設定パネル（M-SET）には移動先の案内のみを残す。

## 7. 認証・起動フロー

```mermaid
sequenceDiagram
    participant U as 利用者
    participant App as App
    participant Auth as Firebase Auth
    participant FS as Firestore

    U->>App: アクセス
    App->>Auth: subscribeAuth()
    Auth-->>App: { ready: true, user, allowed }
    alt 未サインイン
        App-->>U: S-LOGIN 表示
        U->>Auth: Googleサインイン
        Auth-->>App: 認証結果
    end
    alt 許可メール
        App->>FS: tasks / settings / master を購読
        FS-->>App: 初期データ
        App-->>U: S-INPUT 表示（ローディング解除）
    else 非許可メール
        Auth->>Auth: 自動サインアウト
        App-->>U: S-LOGIN（拒否メッセージ）
    end
```

## 8. 遷移一覧表（マトリクス）

| 遷移元＼遷移先 | 入力 | カレンダー | 担当者別 | サマリー | 完了 | マスタ | サインイン |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| サインイン | ○※1 | - | - | - | - | - | - |
| 入力 | ○※2 | ○ | ○ | ○ | ○※3 | ○ | ○※4 |
| カレンダー | ○※5 | ○ | ○ | ○ | ○ | ○ | ○※4 |
| 担当者別 | ○※2 | ○ | ○ | ○ | ○ | ○ | ○※4 |
| サマリー | ○ | ○ | ○ | ○ | ○ | ○ | ○※4 |
| 完了 | ○※5 | ○ | ○ | ○ | ○ | ○ | ○※4 |
| マスタ | ○ | ○ | ○ | ○ | ○ | ○ | ○※4 |

- ※1：許可メールでのサインイン成功時。
- ※2：編集（step/viewpoint/project）・視点/ステップ追加で自画面のフォームへ。
- ※3：視点完了／案件完了／案件中止の確定後。
- ※4：サインアウト操作時。
- ※5：カレンダー/完了のブロック・行クリックで案件編集フォーム（入力画面）へ。

> ナビゲーションバーにより、認証後の6画面は基本的に相互遷移可能。上表の「○」はナビ遷移＋特記の業務遷移を含む。
