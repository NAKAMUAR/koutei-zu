# UI設計原則（品質維持ルール）

このドキュメントは、2026-07 に実施したUIレビュー・大規模改善で導入された設計原則をまとめたもの。
**UI・画面・操作に関わる変更を行う前に必ず読むこと**（CLAUDE.md から参照されている）。
原則の多くは ESLint（`npm run check`、`npm run build` 前に自動実行）で機械的に強制される。
**ESLint のエラーをルール緩和・無効化コメントで回避してはならない。** ルール自体を変えたい場合は、
先にこのドキュメントの該当項目を更新して理由を残すこと。

## 背景：レビュー時の採点と改善の狙い

| 観点 | 改善前 | 改善後の維持目標 | 主な担保手段 |
|---|---|---|---|
| 機能の充実度・業務適合 | 9/10 | 9/10 | 既存機能の挙動を変えない改修プロセス |
| 見た目の統一感 | 7/10 | 7.5/10 | colors パレット・共通部品の使用（原則9） |
| 操作の分かりやすさ | 5.5/10 | 7.5/10 | 原則2・3・4・6 |
| フィードバック・安心感 | 5/10 | 8/10 | 原則1（ESLintで強制） |
| コードの健全性 | 5/10 | 7.5/10 | 原則7・8・10（ESLintで強制） |

この表の「改善後」の水準を**下回らせない**ことが、このドキュメントの目的。

## 原則

### 1. フィードバック：ネイティブダイアログ禁止・削除には「元に戻す」

- `alert()` / `confirm()` / `prompt()` は**使用禁止**（ESLint `no-alert` = error）。
- 確認 → `confirmDialog({ title, message, confirmLabel })`（Promise<boolean>）
- 文字入力 → `promptDialog({ title, message, defaultValue })`（Promise<string|null>）
- 通知 → `notify(message, { type: 'info'|'success'|'error' })`（右下トースト）
- いずれも `useApp()`（src/appContext.js）から取得する。
- **破壊的操作（削除）には可能な限り `notify(..., { undo: () => 復元処理 })` を付ける。**
  タスクの復元は App.jsx の `restoreTasks()` を使う（deletedExternalIds の巻き戻しも行う）。

### 2. 情報設計：ナビは「毎日使うタブ＋集計・帳票グループ」

- 直接表示するタブは毎日使うものだけ：**案件・サマリー・完了・タスクメモ・マスタ**。
- 経理・集計系（帳票・売上登録・案件整理・会社別集計）は **「集計・帳票」NavGroup** に集約。
- 新しい画面を追加するときは「毎日使うか？」を自問し、そうでなければ NavGroup へ入れる。
  直接表示のタブを 6 個以上に増やさない。

### 3. 段階的開示：フォームは「進行管理が1行目・請求系は折りたたみ」

- 新規案件登録フォームのステップ行は、1行目＝種類・制作時間・完了時間（毎回入力するもの）、
  「請求・納品」トグル内＝納品名・依頼日・完了日・金額（あとで入力するもの）。
- 新しい入力項目を足すときは、**毎回の登録で必須でない限り折りたたみ側へ**入れる。
- 値が入っている折りたたみは自動で開く（入力済みが隠れて見えない状態を作らない）。

### 4. 操作の可視性：ツールチップ（title）だけに頼らない

- title 属性はタッチ端末では一切見えない。**クリック・ドラッグできることは見た目で示す。**
- インライン編集できる値 → `className="kz-inline-edit"` を付け、中に
  `<Edit2 size={9} className="kz-pencil" />` を置く（ホバーで鉛筆表示・タッチでは薄く常時表示）。
- ホバー時だけ出す副次操作（削除・▲▼など） → 行コンテナに `kz-row`、対象に `kz-hover-reveal`。
  タッチ端末では常時表示になる（CSS は App.jsx の `<style>` に定義済み）。
- ドラッグ＆ドロップを実装したら、**画面上に凡例テキストを置く**（カレンダー上部の凡例を踏襲）。

### 5. 日時入力：共通 DateTimeField に統一

- 日付＋時刻の入力は `components/common.jsx` の **`DateTimeField`** を使う
  （`value`: 'YYYY-MM-DDTHH:mm'、`defaultTime`、`compact` 対応）。
- `<input type="datetime-local">` は**使用禁止**（ESLint `no-restricted-syntax` = error）。
  入力しづらい環境があるため。時刻だけなら `TimeSelect`、所要時間なら `DurationSelect`。

### 6. モードレス：グローバルモードを追加しない

- 「ONの間だけ挙動が変わる」画面全体のモード（旧・案件編集モード）は**追加しない**。
  切り忘れ・解除忘れ・今どのモードか覚える負担が事故のもと。
- 警告・バナーを一時的に消したいときは、**その警告自体に「今日は表示しない」**を付ける。
  実装パターン：日付キー（`fmtYMD(now)`）付きで localStorage に保存 → 翌日自動復活。
  （実例：InputView の `kz-riskMute` / `kz-lateRiskMute`）

### 7. コード構造：分割を維持し、propsバケツリレーに戻さない

- 配置ルール：画面＝`src/views/`、共通部品＝`src/components/`、純ロジック＝`src/lib/`、
  状態管理とハンドラ＝`App.jsx`、共有コンテキスト＝`src/appContext.js`。
- **1ファイル 1800 行まで**（App.jsx のみ 2600 行。ESLint `max-lines` = error）。
  超えそうなら先に分割する。上限を引き上げてはならない。
- 共有データ（tasks / scheduled / settings / now …）とタスク操作ハンドラは
  **`useApp()` で取得**する。コンポーネント固有の値（group・task・form など）だけを props で渡す。
  ハンドラを2階層以上 props で運ぶ設計に戻さない。

### 8. 遅延読み込み：新しいトップレベルビューは React.lazy

- 「案件」タブ以外のビューは `React.lazy(() => import(...))` で追加する（App.jsx の先頭を踏襲）。
  初回ロードを重くしない。

### 9. スタイル：パレットと共通部品を使う

- 色は `useApp()` の `colors`（bg / surface / border / text / textMute / accent / accentSoft / progress）
  を使う。新しい直書きの色コードを増やさない（意味が同じなら既存色に合わせる）。
- フォントは `fontJP`（本文）・`fontDisplay`（見出し）。
- ボタン等は `components/common.jsx` の既存部品（NavButton / NavGroup / Combobox / TimeSelect /
  DurationSelect / DateTimeField / ConfirmModal / PromptModal / ToastStack …）を先に探す。

### 10. 検証：`npm run check` がビルドのゲート

- `npm run build` は `prebuild` で自動的に `npm run check`（ESLint）を実行する。
  **エラーが1件でもあればビルドは失敗する**（＝デプロイもできない）。これが品質の最終ゲート。
- 挙動を変える改修をしたら、ビルドに加えて `vite preview` ＋ ブラウザでログイン画面が
  描画されることを最低限確認する（モジュールレベルのクラッシュ検出）。

## ESLint で強制されているルール（eslint.config.js）

| ルール | レベル | 対応する原則 |
|---|---|---|
| no-alert | error | 原則1（ネイティブダイアログ禁止） |
| no-restricted-syntax（datetime-local） | error | 原則5（DateTimeField統一） |
| max-lines（1800 / App.jsx 2600） | error | 原則7（分割維持） |
| no-undef / react/jsx-no-undef | error | import忘れ・タイポの検出 |
| react-hooks/rules-of-hooks | error | フック順序バグの防止 |
| no-unused-vars | warn | 死にコードの抑制 |
