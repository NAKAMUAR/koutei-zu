# デザイン改善 3案（2026-07）

対象：案件タブ（進行中一覧）を中心とした全画面のビジュアル刷新。
モックアップは同ディレクトリの `2026-07_design-proposals.html` をブラウザで開いて確認する。

## 現行デザインの課題

- 本文フォントが 10〜11px の箇所が多く小さい
- 薄いグレー文字（#6b6b6b）× 生成り背景（#faf8f3）でコントラストが不足
- 会社ピル・至急バッジ・納期・担当がすべて同じ強さで並び、視線の置き場が定まらない
- カード境界が 1px の薄い罫線のみで、案件が増えると行が溶けて見える

## 案1：和モダン・リーダブル（実装コスト：小）

現行の世界観（生成り × 朱赤 × 明朝ロゴ）を維持したまま可読性だけを立て直す。

- 最小フォント 12px、案件名 16px に引き上げ。文字色を1段濃く（#6b6b6b → #5c5a54 / 見出し #262626）
- 納期を右端に定位置化して数字を大きく表示
- カードに会社色の左帯＋薄い影。会社セクションは下線付き見出しで大きく区切る
- 「至急」は淡色バッジへ。強い色（朱赤）は納期超過だけに絞る
- 実装：App.jsx の `colors` 定数＋各ビューの fontSize / padding 調整が中心。構造変更なし。1〜2日目安

## 案2：クリーンSaaS（実装コスト：中）

背景を淡いブルーグレー、アクセントを青にした業務ツール調。色に「意味」だけを持たせる。

- 状態色を4色に固定：緑=順調 / 橙=至急 / 赤=超過 / 青=進行中。淡色チップで統一
- 会社色ピルは小さな角丸ドットに縮小し、彩度の高い面積を削減
- ナビは下線タブ、サブタブはセグメント切替。ヘッダを 52px に圧縮
- 実装：colors 全面差し替え＋バッジ/チップの共通コンポーネント化。3〜4日目安

## 案3：ダッシュボード型（実装コスト：大）

レイアウトごと刷新。ナビを左の濃色サイドバーへ移し、進行中一覧を「1視点=1行」のテーブルに。

- 納期・担当・進捗が列で縦に揃う（tabular-nums）ため、多数案件の比較・優先判断が速い
- 状態は「進行中・待機・超過・完了」のドット4色。会社はグループ行で区切る
- 実装：InputView / ViewpointList の再構成＋編集UIの再設計が必要。1〜2週間目安

## 推奨

**まず案1を実施し、余力があれば案2の「状態チップ（意味のある4色）」だけを移植する。**
案1は colors 定数と文字サイズの調整だけで効果が大きく、操作を一切変えない。
案3は繁忙期に一覧性の限界を感じたときの次の一手として温存する。
